# Visual Brand System

This document defines the visual direction for `zjavier-next` content, blog visuals, social media assets, image prompts, MDX components, and future video formats.

The goal is to create a recognizable visual system that feels technical, personal, modern, expressive, and independent without becoming generic AI-generated tech branding.

---

## Core Visual Idea

The visual identity should feel like:

- A developer laboratory
- A personal operating system
- A black-and-white technical sketchbook
- A modern creative coding studio
- A small independent tech brand
- A place where code, automation, AI, and visual systems meet

The brand should feel precise, minimal, slightly experimental, and human.

## Core Aesthetic

The primary aesthetic is:

Black and white technical minimalism with orange accents, dot-matrix textures, clean typography, and subtle interface-inspired layouts.

This should not feel like generic SaaS branding.

It should feel more like:

- A personal dev lab
- A technical field notebook
- A digital archive
- A creative coding identity
- A modern independent builder brand

## Color System

### Primary Colors

Use a strong black and white base.

```txt
Black: #050505
White: #F7F7F2
Off-white: #EDEDE6
Soft gray: #A8A8A0
Dark gray: #1A1A1A
```

### Accent Color

Orange should be the main recurring accent.

```txt
Orange: #FF6A00
Soft orange: #FF8A2A
Burnt orange: #C94F00
```

Orange should be used intentionally, not everywhere.

Good uses:

- Small highlights
- Active states
- Key labels
- Dots or signal marks
- Underlines
- Section tags
- Small UI accents
- Call-to-action elements

Avoid using orange as a full background unless the design intentionally needs a high-impact moment.

---

## Secondary Accent Options

Secondary accents can be used sparingly when needed.

```txt
Muted blue-gray: #60717A
Muted green-gray: #6F7D68
Warm beige: #D8C7A3
Soft red: #B84A3A
```

These should not compete with orange.

The system should still read primarily as black, white, gray, and orange.

## Typography Direction

Preferred typography direction:

- Modern
- Rounded but technical
- Clean
- Slightly expressive
- Not overly corporate

Primary brand typography references:

- MuseoModerno for expressive name/logo usage
- Chivo for subtitles, metadata, captions, and practical UI text

Use typography to create contrast:

- MuseoModerno: identity, headers, expressive moments
- Chivo: readable body, captions, technical labels

For generated images, prompts can reference:

```txt
modern rounded geometric typography, inspired by MuseoModerno for titles and Chivo for small technical labels
```

Do not ask image models to reproduce exact fonts unless the tool supports text reliably.

Use text sparingly in generated visuals because generated text can become distorted.

## Blog Cover Direction

Blog covers should be clean and recognizable.

Preferred blog cover structure:

- Black or off-white background
- One strong visual metaphor
- Orange accent
- Subtle grid or dot pattern
- Minimal text or no text
- Technical diagram or abstract interface element

Good blog cover concepts:

- Workflow diagram made of simple nodes
- Dithered object or portrait
- Terminal-like panel with abstract content
- Layered cards representing systems
- Minimal architecture diagram

- Full paragraphs of text inside images
- Generic AI-generated laptops
- Overcrowded dashboards
- Random glowing circuits
- Excessive neon
- Stock-photo realism

## Carousel Direction

Carousels should be useful, simple, and visually consistent.

Recommended structure:

1. Hook slide
2. Problem slide
3. Core idea slide
4. Process or example slide
5. Lesson learned slide
6. CTA or link slide

Visual rules:

- One idea per slide
- Large readable type
- Black/white base
- Orange accent
- Minimal icons
- Consistent footer or small brand mark
- Use diagrams when they clarify the idea

Avoid:

- Too much text
- Tiny code snippets
- Random images
- Inconsistent styles between slides
- Engagement-bait formatting

---

## Screenshot Treatment

Screenshots can be used in blog and social visuals.

Preferred treatment:

- Place screenshot inside a clean frame
- Add subtle border
- Use black or off-white background
- Add small label or caption
- Use orange highlight only for key areas
- Keep annotations minimal

Avoid:

- Over-annotating
- Blurry screenshots
- Low contrast
- Showing sensitive data
- Showing private tokens, emails, or credentials

## Video Direction

1. Quick branded intro
2. Hook or title card
3. Three to five content beats
4. Diagram, screenshot, or code moment
5. Final takeaway
6. Outro with site or handle

Visual style:

- Black/white base
- Orange accents

- Caption-first readability
- Minimal transitions

Avoid:

- Random AI-generated video scenes
- Inconsistent styles between videos
- Overly cinematic stock footage
- Generic tech background loops
- Heavy 3D unless intentionally designed

## Visual Quality Checklist

Before using a generated visual, check:

- Does it feel like the brand?
- Is it mostly black, white, gray, and orange?
- Is the orange accent intentional?
- Is the composition clean?
- Is the text minimal and readable?
- Does it avoid generic AI imagery?
- Does it support the content?
- Does it feel technical but human?
- Would it look coherent next to other brand visuals?
- Is there any private or sensitive information visible?

---

## Prompt Quality Checklist

A good visual prompt should include:

- Format
- Topic
- Style
- Composition
- Mood
- Color direction
- Texture direction
- What to avoid
- Platform or use case
- Aspect ratio when relevant

Avoid prompts that only say:

- make it modern
- make it futuristic
- AI-themed
- tech style
- professional design

Those are too vague and will produce generic results.

---

## What To Avoid Overall

Avoid visual output that feels like:

- Generic AI SaaS
- Cyberpunk cliché
- Corporate stock illustration
- Random glowing network
- Robot mascot branding
- Blue holographic dashboard
- Overly polished but empty
- Too colorful
- Too cluttered
- Too dependent on unreadable generated text

The brand should not feel like every other AI tool.

It should feel like an independent builder's technical lab.

---

## Final Visual Rule

The visual system should make the work feel built, not generated.

AI can help create assets, variations, and prompts, but the final visual direction should preserve taste, restraint, and consistency.
